# Exam Feature — Phase 1.5: Dedicated Exam Engines

**Date:** 2026-04-26
**Branch:** development-branch
**Author:** Justine L. Llamera

---

## Why This Was Built

Phase 1 reused all 27 curriculum engines inside `ExamOrchestrator`. While this worked technically, it created a correctness problem: the curriculum engines are designed to give immediate correct/wrong feedback after every answer — haptic notifications, speech ("Correct!", "Try again!"), badge icons, color flashes, and auto-advance delays. None of that is appropriate in exam context.

Rather than patching the existing engines with `examMode` props (which would add conditional logic throughout shared components), the decision was made to build a dedicated exam engine layer. This gives the exam stack clean separation, its own JSON contract, and a stable foundation for teacher-created exams.

**Scope of Phase 1.5:** Two engine types were chosen based on simplicity of interaction and likelihood of appearing in real teacher-created exams: `NUMPAD` (fill-in-the-blank numeric) and `VISUAL_PICKER` (visual story + multiple choice). Gesture-heavy types (dragdrop, geoboard, shapetracer) are explicitly excluded from the exam stack.

### `src/Components/Game/Exam/Engines/ExamNumpadEngine.jsx`

**Based on:** `src/Components/Game/Curriculum/Engines/NumpadEngine.jsx`

**What was kept:**
- Full numpad grid UI (3×4 bulky keys, backspace key, blinking cursor)
- All tactile press animations (`withSpring` translateY + borderBottomWidth on keys)
- `Haptics.impactAsync(Light)` on key press and backspace — this is tactile feel for the input, not answer feedback
- `Haptics.impactAsync(Medium)` on CHECK button press
- ZoomIn/FadeInDown/FadeInUp entering animations
- Display scale pop animation on key press
- Blinking cursor (`withRepeat` + `withSequence`)
- `blank = 'right' | 'left'` equation rendering
- `maxDigits` cap

**What was removed (feedback behavior):**
| Removed | Why |
|---|---|
| `Haptics.notificationAsync(Success)` on correct | Reveals correctness to student mid-exam |
| `Haptics.notificationAsync(Error)` on wrong | Reveals incorrectness to student mid-exam |
| `setTimeout(() => onResult(true, ...), 600)` | Delay was a UX affordance for the feedback flash — not needed without feedback |
| `setTimeout(() => setInput(''); setIsWrong(false); }, 1000)` | Input-clear loop for retrying wrong answers — not needed; student navigates away to retry |
| `shakeX` + `shakeStyle` + shake `withSequence` animation | Shake is an error signal — removed entirely |
| `isWrong` state | No wrong-state visual exists in exam engine |
| Dynamic instruction color (`Colors.error` / `Colors.success`) | No color feedback on instruction text |
| `answerBoxWrong` style (red border/background) | No error styling on answer box |
| `answerBoxCorrect` style (green border/background) | No correct styling on answer box |
| `answerTextCorrect` color | No green text on correct input |

**What was added:**
- **`questionText` display card** above the equation — curriculum NumpadEngine doesn't render `data.question` (lessons provide context via narration), but exam questions are standalone so the prompt text must be visible
- **`answerBoxAnswered` style** — neutral `Colors.outlineVariant` solid border after submission (replaces the dashed primary border), signals "locked in" without implying correctness
- **`label` prop on `CheckButton`** — changes to `'ANSWERED ✓'` after submission (neutral disabled styling)

**Answer behavior in exam mode:**
1. Student types digits → display pops (animation)
2. Student taps CHECK → `onResult(isCorrect, [input])` fires immediately, `setAnswered(true)`
3. Button shows `'ANSWERED ✓'` (disabled), answer box border changes to neutral
4. To change answer: student navigates to another question and back — `key={currentIndex}` remount resets engine to blank state

### `src/Components/Game/Exam/Engines/ExamVisualPickerEngine.jsx`

**Based on:** `src/Components/Game/Curriculum/Engines/VisualPickerEngine.jsx`

**What was kept:**
- Full story-header layout: asset card (image) + speech bubble (question text)
- Equation pill (`addends[0] operator addends[1] = ?`)
- Option buttons with `ZoomIn.springify().delay(index * 80)` entering animation
- Tactile press animations on option buttons (`withSpring` translateY + borderBottomWidth)
- `Haptics.impactAsync(Light)` on option press — tactile selection feel, not answer feedback
- `Haptics.impactAsync(Medium)` on CHECK button press
- Shuffled options (deterministic per mount via `useMemo`)
- `AssetDisplay` for character/illustration rendering
- `speechBubbleFull` layout variant (when no assetId)
- CHECK button full-width with `Colors.tertiary` active / disabled styling

**What was removed (feedback behavior):**
| Removed | Why |
|---|---|
| `evaluation` state (`null` / `'correct'` / `'wrong'`) | Entire evaluation pipeline removed — exam shows no correct/wrong |
| `speechManager.speakInstruction(questionText)` | Speech instruction reveals nothing but was paired with feedback speech; removed for clean separation |
| `speechManager.speakFeedback('Correct!', true)` | Feedback speech — reveals correctness mid-exam |
| `speechManager.speakFeedback('Try again!', false)` | Feedback speech — reveals incorrectness mid-exam |
| `Haptics.notificationAsync(Success)` on correct | Reveals correctness |
| `Haptics.notificationAsync(Error)` on wrong | Reveals incorrectness |
| `setTimeout(() => onResult(true, ...), 800)` | Delay was for feedback flash affordance |
| `setTimeout(() => onResult(false, ...), 1000)` | Delay was for feedback flash affordance |
| `evaluation === 'correct'` option color (green) | No correct-state color on options |
| `evaluation === 'wrong'` option color (red) | No wrong-state color on options |
| Checkmark badge icon on correct option | No badge icons in exam mode |
| Close-circle badge icon on wrong option | No badge icons in exam mode |
| `speechManager` import | No longer needed |

**What was added:**
- **`label` prop on `CheckButton`** — changes to `'ANSWERED ✓'` after resolving
- **`resolved` state replaces `resolved + evaluation`** — since there are only two states (not resolved / resolved), no evaluation enum is needed

**Answer behavior in exam mode:**
1. Student taps an option → button highlights in `Colors.secondaryContainer` (selected state only — no correct/wrong color)
2. Student taps CHECK → `onResult(isCorrect, [String(selectedOption)])` fires immediately, `setResolved(true)`
3. Button shows `'ANSWERED ✓'` (disabled), options lock (no further selection)
4. To change answer: navigate away and back — remount resets to blank

## Architectural Invariants Added

### Dedicated exam engine contract
All exam engines in `src/Components/Game/Exam/Engines/` must:
- Accept `{ data, onResult }` props — same surface as curriculum engines
- Call `onResult(isCorrect, [userAnswer])` **immediately** on CHECK (no setTimeout)
- Show **no correct/wrong feedback** (no color change, no badge icon, no speech, no haptic notification)
- Show a neutral **"ANSWERED ✓"** state after submission (button label change + disabled)
- Retain **tactile press haptics** (`Haptics.impactAsync`) for key/button presses — these are input feel, not answer feedback

### Remount-based answer change
Exam engines do NOT need to support in-place answer changing. `key={currentIndex}` on the engine in `ExamOrchestrator` guarantees a full remount whenever the student navigates to a different question and comes back. The engine always starts clean. The `answers` map in `ExamOrchestrator` remembers the previous answer and will overwrite it if the student re-submits.
