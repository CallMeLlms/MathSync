# Timeline Log: Engine Crash and Generator Fixes

**Timestamp**: 2026-04-15 23:41:00  
**Topic**: Critical Stability Fixes for Generative Engines

## Summary of Changes

### 1. PlaceValueEngine.jsx (Stability)
- **Resolved UI Thread Crash**: Wrapped the `onSelect` callback in `runOnJS` within the `Gesture.Tap()` handler. This prevents a native crash that occurs when updating React state or triggering haptics from the UI thread.
- **Implemented Submit Guard**: Added a `hasAnswered` `useRef` to prevent double-submissions. This ensures that rapid taps on the "Submit" button do not trigger multiple `onAnswer` calls or double-scoring.

### 2. placeValueGenerator.js (Logic & Performance)
- **Infinite Loop Guard**: Added `MAX_ATTEMPTS = 10` to the `compare` problem generator. If a unique second number isn't found within 10 tries, the generator now uses a "forced resolution" nudging logic to guarantee a valid problem.
- **Comparison Range Enforcement**: Added `compareMin` logic to ensure that comparison problems for hundreds are not trivially compared against 2-digit numbers, maintaining pedagogical quality.

## Impact
- **CRITICAL**: The app no longer crashes when a user interacts with choice tiles.
- **Improved UX**: Prevents UI freezes and prevents double-scoring bugs.
- **Higher Quality Content**: Comparison problems are now more consistent in difficulty.
